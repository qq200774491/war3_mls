# mls-sim.exe 使用说明

## 启动

```powershell
mls-sim.exe --script-dir "D:/你的云脚本目录/script"
```

启动后自动创建房间、加载 `main.lua`、打开浏览器控制面板。

## 命令行参数

| 参数 | 缩写 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `--host` | 无 | `127.0.0.1` | 监听地址 |
| `--port` | `-p` | `5000` | 监听端口 |
| `--script-dir` | `-s` | 无 | 云脚本目录（目录下需有 `main.lua`） |
| `--config` | 无 | `config.json` | 配置文件路径 |

## 用法示例

```powershell
# 最简启动
mls-sim.exe -s "D:/Code/MlScriptFolder/script"

# 换端口
mls-sim.exe -s "D:/script" -p 5001

# 用配置文件（可预设玩家、道具、存档）
mls-sim.exe --config "D:/my_config.json"

# 空启动，在 Web 面板手动建房
mls-sim.exe
```

## Web 控制面板

启动后浏览器自动打开 `http://127.0.0.1:5000`：

- 左侧：房间列表，点击切换
- 右上：启动 / 停止 / 销毁房间
- 玩家卡片：查看状态，模拟断线 / 重连 / 退出
- 发送事件：填事件名和数据，选目标玩家，点发送
- 日志面板：实时日志，支持级别过滤和搜索
- 出站事件：云脚本发给客户端的事件
- 状态查看：房间完整 JSON，点刷新

## 本地自动化调试

除 Web 控制面板外，模拟器还提供 `/api/debug/*` HTTP 接口，方便脚本、LLM 或外部工具自动读取日志和控制房间：

| 能力 | 接口 | 说明 |
| --- | --- | --- |
| 读取房间日志 | `GET /api/debug/rooms/{room_id}/logs` | 从房间侧日志缓存读取日志，可用 `limit`、`level`、`q`、`since` 筛选。 |
| 清空房间日志 | `POST /api/debug/rooms/{room_id}/logs/clear` | 清空指定房间的房间侧日志缓存，不清空 Web 控制面板本地显示列表。 |
| 重启房间 | `POST /api/debug/rooms/{room_id}/restart` | 基于旧房间的脚本目录、模式和玩家快照创建新房间，并返回新的房间 ID。 |
| 服务级重启 | `POST /api/debug/service/restart` | 模拟器进程内不支持，返回 `501`；需要外部工具重新拉起进程。 |

这些接口只面向本地开发调试，不包含线上运营鉴权、多用户权限、审计日志或访问令牌轮换。不要把 `/api/debug/*` 暴露到公网。

## 存档

- 房间停止时自动保存到 `archives/` 目录
- 下次创建同名脚本房间时自动读取上次存档
- 直接关进程不会保存，需要先点"停止"

## 常见问题

| 问题 | 解决 |
| --- | --- |
| 端口占用 | `--port 5001` 换端口 |
| 浏览器没打开 | 手动访问 `http://127.0.0.1:5000` |
| 脚本没加载 | 检查目录下有没有 `main.lua` |
| 存档没保存 | 先点"停止"再关程序 |
| 日志 LOG FUSE | 日志太频繁被限流，减少输出 |
