# 快速开始

## 环境要求

- Rust 工具链（[rustup](https://rustup.rs/) 安装，选 stable 即可）
- Windows 10 或更高版本

## 编译

```powershell
cd mls-sim-rs
cargo build --release
```

产物位于 `target/release/mls-sim.exe`，约 5MB，不依赖任何外部 DLL。

## 启动

### 最简启动（指定脚本目录）

```powershell
mls-sim.exe --script-dir "D:/你的脚本目录/script"
```

启动后：
- 自动创建一个房间
- 加载脚本目录下的 `main.lua`
- 打开浏览器访问 Web 控制面板

### 使用配置文件

```powershell
copy config.example.json config.json
# 编辑 config.json，修改 script_dir 等路径
mls-sim.exe
```

### 空启动（手动建房）

```powershell
mls-sim.exe --port 5000
```

打开 `http://127.0.0.1:5000`，在 Web 面板里点"+ 新建"手动创建房间。

## 验证是否启动成功

```powershell
curl http://127.0.0.1:5000/api/health
```

正常返回：

```json
{
  "ok": true,
  "name": "mls-sim",
  "version": "0.3.0",
  "room_count": 1,
  "rooms": [...]
}
```

## 发送测试事件

通过 curl 向云脚本发送事件：

```powershell
curl -X POST http://127.0.0.1:5000/api/rooms/room-001/events -H "Content-Type: application/json" -d "{\"ename\":\"testapi\",\"evalue\":\"\",\"player_index\":0}"
```

也可以在 Web 控制面板的"发送事件"区域直接操作。

## 模拟客户端轮询

```powershell
curl http://127.0.0.1:5000/api/bridge/poll/room-001/0
```

返回云脚本发给该玩家的所有待取事件。

## 自动化调试

本地脚本或外部工具可以通过调试接口读取房间日志、清空房间日志缓存和重启房间：

```powershell
curl "http://127.0.0.1:5000/api/debug/rooms/room-001/logs?limit=50"
curl -X POST http://127.0.0.1:5000/api/debug/rooms/room-001/logs/clear
curl -X POST http://127.0.0.1:5000/api/debug/rooms/room-001/restart
```

服务级重启不由模拟器进程内接口承担，`POST /api/debug/service/restart` 会返回 `501`。

本地调试接口的回归测试位于 `mls-sim-rs/tests/debug_api.rs`。

## 命令行参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--host` | `127.0.0.1` | 监听地址 |
| `--port` / `-p` | `5000` | 监听端口 |
| `--script-dir` / `-s` | 无 | 云脚本目录，启动后自动建房 |
| `--config` | `config.json` | 配置文件路径 |
